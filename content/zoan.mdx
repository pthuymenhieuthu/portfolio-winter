## Phase 1: Understanding the Problem

The platform had grown rapidly, which led to visual fragmentation.  
There were mismatched components, inconsistent spacing, conflicting color usage, and no unified design language between desktop and mobile.

I mapped the entire system, identifying issues in structure, hierarchy, and consistency.  
This audit became the foundation of the redesign.

## Phase 4: Applying the System Across Platforms

### Task Flow Mapping

Before designing any screen, I created detailed task flows to map how users moved through the system from start to end.  
This included both happy paths and potential edge cases, ensuring the experience stayed stable even under unexpected scenarios.

These flows aligned the entire team around the same mental model and made it easier to break work into manageable increments for later sprints.  
By the time the UI phase began, we had already validated the structure of the experience.

### Desktop Platform
